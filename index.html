Abstract

Many networked control systems rely on the User Datagram Protocol (UDP) to achieve low-latency communication between a centralized controller and multiple devices. These systems typically exchange bidirectional traffic, consisting of controller-to-device control messages and device-to-controller telemetry data, each with different Quality of Service (QoS) requirements. Because UDP provides no built-in congestion control or reliability, such systems are vulnerable to increased delay and packet loss when deployed on shared networks.

Project Description

This project proposes to evaluate the effectiveness of packet scheduling algorithms in enforcing QoS for bidirectional UDP-based control and telemetry traffic. Using the ns-3 network simulator, a simple client–server topology will be modeled in which a centralized orchestrator communicates with multiple devices over a shared bottleneck link in the presence of background traffic. Packet scheduling disciplines such as First-In First-Out (FIFO), Priority Queuing, and Weighted Fair Queuing will be compared.

Performance metrics including end-to-end delay, jitter, packet loss, and throughput will be used to assess how well each scheduling approach supports the QoS requirements of control and telemetry traffic.
